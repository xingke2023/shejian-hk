## 二、模块划分 & API 设计

### 模块 1：AI 店长助手（核心入口）

**功能：**
- 文字 / 拍照 / 语音输入（碎片化信息录入）
- AI 解析 → 结构化处理 → 自动分发到对应模块
- 异常录入检测与提醒
- AI 实时经营建议

**前端页面：**
- `/assistant` — AI 对话主界面（类聊天窗口）
- 支持文字输入、图片上传、语音录入

**后端 API：**
```
POST   /api/ai/message              # 发送消息（文字/图片）
POST   /api/ai/voice                # 语音上传转文字
GET    /api/ai/sessions             # 会话历史
GET    /api/ai/sessions/{id}        # 单条会话详情
POST   /api/ai/sessions/{id}/parse  # 手动触发AI解析分发
```

**数据表：** `ai_sessions`, `ai_messages`, `ai_command_templates`

---

### 模块 2：商品库与库存系统

**功能：**
- 商品标准化管理（参考银豹字段）
- 供应商资料管理（结算方式、供货周期）
- 商品与供应商绑定
- 进货入库（支持语音/微信群信息解析）
- 库存实时变动、品质反馈
- 重点品类强制上报、语音盘点
- AI 智能库存核算（期初库存 - 销售量 - 损耗）

**前端页面：**
- `/products` — 商品库列表/详情
- `/suppliers` — 供应商管理
- `/inventory` — 库存看板
- `/inventory/purchase` — 进货入库
- `/inventory/stocktake` — 盘点上报
- `/inventory/alerts` — 库存预警

**后端 API：**
```
# 商品
GET    /api/products                # 商品列表
POST   /api/products                # 新增商品
PUT    /api/products/{id}           # 更新商品
DELETE /api/products/{id}           # 删除商品
GET    /api/product-categories      # 分类列表

# 供应商
GET    /api/suppliers               # 供应商列表
POST   /api/suppliers               # 新增供应商
PUT    /api/suppliers/{id}          # 更新供应商
GET    /api/suppliers/{id}/products # 供应商商品

# 进货/库存
GET    /api/inventory               # 库存列表
POST   /api/purchase-orders         # 创建进货单
PUT    /api/purchase-orders/{id}    # 更新进货单
POST   /api/inventory/receive       # 到货确认（支持AI解析）
POST   /api/inventory/feedback      # 品质/实况反馈
POST   /api/inventory/stocktake     # 盘点上报
GET    /api/inventory/alerts        # 库存预警
POST   /api/inventory/adjust        # 人工调整
```

**数据表：** `products`, `product_categories`, `suppliers`, `supplier_products`, `inventory`, `inventory_transactions`, `purchase_orders`, `purchase_order_items`, `inventory_count_sheets`

## 四、AI 处理流程

```
用户输入（文字/图片/语音）
        │
        ▼
   AI解析层（Claude API）
        │
   ┌────┴─────────────────────────────┐
   │ 意图识别                          │
   │ - 进货上报 → 库存模块             │
   │ - 品质反馈 → 库存模块             │
   │ - 促销建议 → 促销模块             │
   │ - 竞品上报 → 竞品模块             │
   │ - 支出录入 → 财务模块             │
   │ - 经营咨询 → AI建议               │
   └──────────────────────────────────┘
        │
        ▼
   结构化数据写入对应模块
        │
        ▼
   实时返回AI建议 + 操作确认
```

## 五、技术实现要点

### 后端（Laravel）
- **AI集成**：使用 Claude API，封装 `AiService` 统一调用
- **队列**：耗时AI任务放入 Laravel Queue（预测、批量解析）
- **定时任务**：促销提醒、库存预警用 `php artisan schedule:run`
- **文件存储**：票据/图片用 S3 或本地 storage
- **中间件**：`StoreScope` 中间件自动过滤门店数据

### 前端（Next.js）
- **状态管理**：Zustand 管理全局门店/用户状态
- **实时更新**：关键看板数据轮询或 WebSocket
- **移动优先**：所有上报页面适配手机端
- **企业微信**：通过企业微信 JS-SDK 调用摄像头/语音
